Ben's git blog

Git is a very useful tool to track the changes you make. By utilizing version control, git makes it easy to see the history of a file through the various snapshots that are taken each time a file is edited. You can do this by through commits. Commits are an action or collection of actions that are taken to modify a file. Then those actions are committed, the current version as well as the previous versions are available to view. It allows you to see what you've changed throughout the history of the file.

While you're in the terminal and making changes, git status is a useful command to show you what, if any, files have been modified or are untracked. It also tells you what you need to do to move those files to a commit or what the next stage is for those files.

David's display blog

When writing CSS code, we use the property 'display' which helps determine how an element is displayed on the page. From here, we specify whether we want whatever we're styling to be inline, block, or inline-block and with proper syntax looking like this: [display: inline;] or [display: inline-block;].

Visually, when displaying an inline element, it won't look like much, but having multiple elements together, you'll see they are viewed next to one another. This is because an inline element does not affect anything but what's between the tag itself.
